9.1 Introduction

An effective system combines elements into an architecture, or design, that is flexible, cost-effective, technically sound, and able to support the information needs of the business

System architecture translates the logical design of an information system into a physical structure that includes hardware, software, network support, and processing methods

9.2 System Architecture

System architecture is the overall design of the system, including hardware, software, and network components

The architecture must be flexible enough to accommodate future growth and change

The architecture must also be cost-effective, technically sound, and able to support the information needs of the business

9.3 System Architecture

System Architecture Checklist
  • Enterprise Resource Planning (ERP)

  • Initial Cost and TCO

  • Scalability

  • Web Integration

  • Legacy System Interface Requirements

  • Processing Options

  • Security Issues

9.5 Processing Methods

Online processing Characteristics
  1. The system processes transactions completely when and where they occur

  2. Users interact directly with the information system

  3. Users can access data randomly

  4. The information system must be available whenever necessary to support business functions

9.5 Processing Methods

Batch Processing
In a batch processing system, data is collected and processed in groups, or batches

The IT operations group can run batch programs on a predetermined schedule without user involvement; and Batch programs require significantly fewer network resources than online systems

9.5 Processing Methods

Combined Online and Batch Processing

Online processing offers an inherent advantage because data is entered and validated as it occurs

Online processing is more expensive to implement and maintain than batch processing

Backup and recovery for online processing is more difficult

In many situations, batch processing is cost-effective, less vulnerable to system disruption, and less intrusive

9.6 System Management and Support

Performance Management

Performance management is the process of measuring and monitoring system performance

Performance management tools monitor system performance and provide alerts when performance falls below acceptable levels

9.6 System Management and Support

Fault Management, Backup, and Disaster Recovery
Includes monitoring the system for signs of trouble, logging all system failures, diagnosing the problem, and applying corrective action

Backup and recovery procedures are essential components of system management

Another common strategy is to use a RAID (redundant array of independent disks) system

9.7 System Design Specification

The system design specification is a comprehensive document that describes the recommended technical architecture for the information system
  1. Executive summary

  2. System components

  3. System environment

  4. Implementation requirements

  5. Time and cost estimates

  6. Appendices

9.7 System Design Specification

User Approval
Users must review and approve the interface design, report and menu designs, data entry screens, source documents, and other areas of the system that affect them

When the system design specification is complete, you distribute the document to a target group of users, IT department personnel, and company management

9.7 System Design Specification

Presentation
The presentations give you an opportunity to explain the system, answer questions, consider comments, and secure final approval

The first presentation is to the systems analysts, programmers, and technical support staff members

Management might reach one of three decisions: proceed with systems development, perform additional work on the systems design phase, or terminate the project
