3.1 Introduction

Systems analysis is the second of five phases in the systems development life cycle (SDLC)

The overall objective is to understand the proposed project, ensure that it will support business requirements, and build a solid foundation for system development

The systems analyst is the person who performs systems analysis

Use models and other documentation tools to visualize and describe the proposed system

3.2 System Analysis Activities

  • Requirements Modelling
  • Object Modelling
  • Data and Process Modelling
  • Development Strategies

3.4 System Requirements Checklist

  Input
  Output
  Processes
  Performance
  Controls

3.4.1 Input

Guests/Customers must register into the website to book/reserve rooms. But they can search rooms without registering.

When registering, the guest/customer must input email, password and other details (E.g., First Name, LastName ang Passport/IC no.)

The receptionist/employee must log-in their credentials before they can view or manage any booking/reservations

3.4.2 Output

The Web site must report online volume statistics every four hours, and hourly during peak periods

The hotel booking management information system must manage the booking/reservation of the guest/customers. This includes the check-in/check-out, cancelled booking, booked rooms, available rooms.

The manager must generate various reports needed for decision-making

3.4.3 Processes

The receptionist/employee must check-in or check-out the guest/customers

> The receptionist/employee must process the payments of the guest/customers through the hotel booking management information system

3.4.4 Performance

The system must support hundreds of users online simultaneously

Response time must not exceed four seconds

Booking/Reservation confirmation must be sent to the guest/customer within 5-10 minutes.

3.4.5 Controls

The system must provide log-in security for guest/customer when booking

The system must provide log-in security for receptionist/employee when managing booking/reservation

An employee record must be added, changed, or deleted only by the manager/admin

A customer record must change, or deleted only by receptionist/employee

3.5 Fact-Finding Techniques

Fact-finding is the process of collecting information about the existing system and the requirements for the new system

Who, What, Where, When, How, and Why?

  Interviews
  Document Review
  Observation
Questionnaires & Surveys
  Research

Lesson 3 Summary

  • The systems analysis phase includes three activities: requirements modeling, data and process modeling, and consideration of development strategies

  • The main objective is to understand the proposed project, ensure that it will support business requirements, and build a solid foundation for the systems design phase

  • The fact-finding process includes interviewing, document review, observation, questionnaires, sampling, and research

  • Systems analysts should carefully record and document factual information as it is collected, and various software tools can help an analyst visualize and describe an information system
