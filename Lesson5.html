5.3 Object Modeling with the UML

The UML uses a set of symbols to represent graphically the various components and relationships within a system

It mainly is used to support object-oriented system analysis and to develop object models

5.4 Use-Case Modelling

A use case is a sequence of actions that provide something of measurable value to an actor and is drawn using a use-case diagram

Symbols - Actor, Use Case, System Boundary, Association

When you identify use cases, try to group all the related transactions into a single use case

After you identify the system boundary, you place the use cases on the diagram, add the actors, and show the relationships

5.5 Class Diagram

A class diagram is a static model that shows the classes of objects in a system and the relationships among them

Each class appears as a rectangle, with the class name at the top, followed by the class's attributes and methods


Example of Class Diagram

5.6 Sequence Diagram

A sequence diagram is a dynamic model that shows the interaction of objects arranged in a time sequence

It includes Classes, Lifelines, Messages and Focuses


Example of a Sequence Diagram

5.7 State Transition Diagram

A state transition diagram is a dynamic model that shows the different states of an object and the transitions that cause a change from one state to another

It includes States, Transitions, Events and Actions


Example of a State Transition Diagram

5.8 Activity Diagram

Activity diagrams are dynamic models that show the flow of control from activity to activity within a system

It includes Initial State, Final State, Activities, Transitions, Decision Nodes and Merge Nodes


Example of Activity Diagram

Lesson 5 Summary

  • Object-oriented analysis focuses on objects and data rather than processes

  • The UML is a standard set of symbols used to model object-oriented systems

  • Use-case diagrams show the interactions between users and the system

  • Class diagrams show the static structure of a system

  • Sequence diagrams show the interactions of objects arranged in a time sequence

  • State transition diagrams show the different states of an object and the transitions that cause a change from one state to another
