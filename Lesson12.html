12.1 Introduction

Systems Operation, Support, and Security is the final phase in the systems development life cycle (SDLC)

The systems operation, support, and security begins when a system becomes operational and continues until the system reaches the end of its useful life

After delivering the system, the analyst has two other important tasks: he or she must support users and provide necessary maintenance to keep the system operating properly

12.3 Maintenance Activities

Maintenance is the process of modifying a system after it has been put into production to correct errors, meet new requirements, or improve processing efficiency

The systems operation, support and security phase is an important component of TCO (total cost of ownership) because ongoing maintenance expenses can determine the economic life of a system

Four types of maintenance task can be identified:

Corrective Maintenance
Adaptive maintenance
Perfective maintenance
Preventive maintenance

12.4 Managing Systems Support

In many companies, systems review committee separates maintenance requests from new systems development requests

Many IT managers believe that evaluating all projects together leads to the best possible decisions

Configuration Management

Maintenance Releases

Version Control

12.5 Managing System Performance

System performance is a measure of how well a system is working

Performance metrics are standards of measurement that are used to assess how well a system is performing

Common performance metrics include:
  • Response time
  • Throughput
  • Availability
  • Accuracy
  • Scalability
  • Reliability

12.6 System Backup and Security

System backup is the process of making duplicate copies of system data that can be used to restore the data if the original data are lost or damaged

Security is the process of preventing unauthorized access to a system's hardware, software, and data

A disaster recovery plan (DRP) is a detailed process for recovering information or a system in the event of a catastrophic disaster
